# Release v0.2.0 — Memory Provider API (RFC 0007)

## Summary

- **Memory Provider API** — loosely coupled; passport core stays identity + permissions
- `local-vault` reference provider under `~/.ai-passport/memory/local-vault/`
- Grant extension: optional `memory` field (`provider_id`, `namespaces`, `mode`)
- `PassportContext.memory` — auto-enriched on `export()` when grant allows
- CLI: `memory init`, `memory status`, `memory store`, `grant --memory <namespaces>`
- MCP: `get_memory_context` + memory block in `get_passport_context`
- RFC 0007 **Accepted**

## Install

```bash
npm install -g @ai-passport-core/cli@0.2.0
```

## Quick start

```bash
ai-passport memory init
ai-passport memory store preferences "User prefers TypeScript" --confidence 0.95
ai-passport grant cursor --memory preferences,projects --yes
ai-passport export cursor
```

## MCP (Cursor)

After global install, restart MCP. New tool: `get_memory_context` (namespace-scoped, grant-enforced).

## Non-goals (this release)

- Knowledge graph, confidence decay — v0.3+
- Mission/tagline change — deferred
- SDK republish — CLI only; `@ai-passport-core/sdk` remains `@0.1.0`
